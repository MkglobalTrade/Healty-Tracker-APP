# Health Command Center — Development Guide

## Local Development Setup

### 1. Install Dependencies
```bash
pnpm install
```

### 2. Build the Frontend
```bash
pnpm build
```

### 3. Set Up Environment Variables
Create a `.env.local` file in the project root:
```bash
cp .env.local.example .env.local
```

Then add your Anthropic API key:
```
ANTHROPIC_API_KEY=sk-ant-your-key-here
```

Get your API key from: https://console.anthropic.com/

### 4. Run the Development Server
```bash
pnpm start
```

The app will be available at `http://localhost:5173`

## Features

### Dashboard
- **Overall Status**: Aggregates glucose, blood pressure, and lab results into a single health indicator
- **Latest Readings**: Shows your most recent glucose and blood pressure measurements
- **Trend Charts**: 7-day line charts for glucose and blood pressure trends with clinical reference ranges
- **Medication Schedule**: Day/Night split view of your current medications

### Upload Labs
- **AI Analysis**: Upload PDF or image of lab reports
- **Auto-Extraction**: Claude AI extracts values, dates, and clinical status
- **Status Indicators**: Green (normal), Yellow (watch), Red (critical)
- **Automatic Integration**: Glucose and BP values feed into trend charts

### Medications
- **Add Medications**: Name, dosage, and frequency (morning, night, or twice daily)
- **Schedule View**: Organized by time of day
- **Easy Management**: Add or remove medications with one click

### History
- **Complete Log**: All readings, uploads, and entries in chronological order
- **Download Report**: Generate HTML report with all your health data (printable to PDF)
- **Full Audit Trail**: Every entry includes date, type, and source

### AI Doctor Chat
- **Context-Aware**: Chat has access to your latest glucose, BP, labs, and medications
- **Clinical Analysis**: Ask questions like "Is my glucose trend improving?" or "Which values concern you most?"
- **Message History**: Maintains conversation context across multiple messages
- **Real-Time Analysis**: Uses Claude Sonnet 4 for accurate health insights

### Health News
- **Auto-Refresh**: Updates every 3 hours while the app is open
- **Curated Topics**: Longevity research, glucose management, cardiovascular health, medical breakthroughs
- **Topic Filtering**: Color-coded by category for quick scanning
- **Web Search Integration**: Claude searches the web for latest health news

## Data Storage

All data is stored in **browser localStorage** under the key `mk-health-center-v1`. This means:
- ✅ Your data stays on your device
- ✅ No server storage required
- ✅ Works offline (except AI features)
- ⚠️ Clearing browser data will erase everything

## API Integration

### Claude API Endpoint
The app communicates with Claude via `/api/claude` endpoint:
- **Method**: POST
- **Body**: Standard Anthropic API message format
- **Response**: Standard Anthropic API response format

### Local Development
When running locally with `pnpm start`, the server:
1. Builds the frontend to `dist/`
2. Serves static files from `dist/`
3. Proxies `/api/claude` requests to Anthropic API
4. Keeps your API key secure on the server

### Production (Vercel)
The app uses Vercel serverless functions (`/api/claude.js`) to keep your API key secure.

## Troubleshooting

### "ANTHROPIC_API_KEY is not configured"
- Make sure you've set `ANTHROPIC_API_KEY` in `.env.local`
- Restart the server after adding the key
- Verify the key is valid at https://console.anthropic.com/

### Chat/News not working
- Check browser console for errors (F12)
- Verify API key is set correctly
- Check that `/api/claude` endpoint is accessible

### Data not persisting
- Check browser localStorage (DevTools > Application > Local Storage)
- Ensure you're not in private/incognito mode
- Clear browser cache if data seems corrupted

### Large file uploads failing
- PDFs/images larger than ~3MB may fail due to serverless limits
- Compress images or use text-based PDFs
- Try uploading smaller files first

## Contributing

To modify the app:
1. Edit `src/App.jsx` (all components are in this file)
2. Run `pnpm build` to rebuild
3. Restart `pnpm start` to see changes
4. Test all features before committing

## Deployment to Vercel

1. Push code to GitHub
2. Go to vercel.com and import the repository
3. Add `ANTHROPIC_API_KEY` to environment variables
4. Deploy!

Every push to GitHub will automatically redeploy the app.
